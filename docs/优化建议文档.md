## 一、快速见效的优化（低投入，高收益）

### 1.1 安装 nodemon 热重载

**问题**: 每次修改代码后需手动 `Ctrl+C` 终止进程再 `node app.js` 重启，开发效率低。

**解决方案**:

```bash
npm install --save-dev nodemon
```

修改 `package.json` 的 `scripts`:

```json
"scripts": {
  "start": "node app.js",
  "dev": "nodemon app.js"
}
```

之后开发时运行 `npm run dev` 即可自动监听文件变化重启。

**改动文件**: `package.json`

---

### 1.2 端口使用环境变量

**问题**: `app.js` 中端口 `3000` 硬编码，部署到云端（如 Render、Railway）时端口由平台动态分配，硬编码会导致服务无法启动。

**解决方案**:

```javascript
const port = process.env.PORT || 3000;
```

**改动文件**: `app.js`

---

**改动文件**: 新增 `routes/todo.js`，修改 `app.js`

### 2.3 Todo 数据持久化

**问题**: 当前数据存储在内存数组 `let todos = []` 中，服务重启后数据全部丢失。

**解决方案**: 使用 Node.js 内置 `fs` 模块，将数据持久化到本地 JSON 文件。

```javascript
const fs = require("fs");
const DATA_FILE = path.join(__dirname, "..", "data", "todos.json");

function loadTodos() {
  try {
    return JSON.parse(fs.readFileSync(DATA_FILE, "utf-8"));
  } catch {
    return [];
  }
}

function saveTodos(todos) {
  fs.writeFileSync(DATA_FILE, JSON.stringify(todos, null, 2), "utf-8");
}
```

**改动文件**: 新增 `data/` 目录，修改 `routes/todo.js`

### 3.2 全局错误处理中间件

**问题**: 每个路由都重复编写 `try-catch` 和 `res.status(500).json(...)`，代码冗余。

**解决方案**: 在 `app.js` 末尾添加统一错误处理中间件：

```javascript
// 全局错误处理中间件
app.use((err, req, res, next) => {
  console.error("未捕获错误:", err);
  res.status(500).json({
    success: false,
    error: err.message || "服务器内部错误",
  });
});
```

路由层可以简化为：

```javascript
router.get("/:city", async (req, res, next) => {
  // 不再需要 try-catch，异常自动传给 next()
  const data = await getWeatherByCity(req.params.city);
  if (!data) return res.status(404).json({ ... });
  res.json({ success: true, data });
});
```

**改动文件**: `app.js`、`routes/weather.js`、`routes/translate.js`

---

### 3.3 封装 asyncHandler 工具函数

**问题**: 即使有全局错误中间件，每个路由仍需手动确保 async 函数中的异常传递给 `next()`。

**解决方案**: 创建 `utils/asyncHandler.js`:

```javascript
// utils/asyncHandler.js
module.exports = function asyncHandler(fn) {
  return (req, res, next) => {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
};
```

路由使用：

```javascript
const asyncHandler = require("../utils/asyncHandler");

router.get(
  "/:city",
  asyncHandler(async (req, res) => {
    const data = await getWeatherByCity(req.params.city);
    res.json({ success: true, data });
  }),
);
```

**改动文件**: 新增 `utils/asyncHandler.js`，修改 `routes/*.js`

### 3.4 引入 express-validator 进行参数校验

**问题**: 路由层手动编写参数校验逻辑（如空字符串判断），分散且容易遗漏。

**解决方案**:

```bash
npm install express-validator
```

```javascript
const { body, validationResult } = require("express-validator");

router.post("/", body("q").trim().notEmpty().withMessage("请输入待翻译的文本"), body("target").notEmpty().withMessage("请选择目标语言"), async (req, res) => {
  const errors = validationResult(req);
  if (!errors.isEmpty()) {
    return res.status(400).json({ success: false, error: errors.array()[0].msg });
  }
  // ...
});
```

**改动文件**: `routes/translate.js`、`routes/weather.js`

## 四、测试与质量保障

### 4.1 添加 API 自动化测试

**问题**: 项目无任何测试，修改代码后无法快速验证功能是否正常。

**解决方案**: 使用 `jest` + `supertest` 为所有 API 端点编写测试。

```bash
npm install --save-dev jest supertest
```

新建 `tests/api.test.js`:

```javascript
const request = require("supertest");
const app = require("../app"); // 需要将 app 导出

describe("GET /api/weather/:city", () => {
  test("应返回北京天气数据", async () => {
    const res = await request(app).get("/api/weather/北京");
    expect(res.statusCode).toBe(200);
    expect(res.body.success).toBe(true);
    expect(res.body.data).toHaveProperty("city");
    expect(res.body.data).toHaveProperty("temperature");
  });

  test("空城市名应返回 400", async () => {
    const res = await request(app).get("/api/weather/ ");
    expect(res.statusCode).toBe(400);
    expect(res.body.success).toBe(false);
  });
});
```

> ⚠️ 注意：需将 `app.listen` 与 `app` 导出分离，测试时不启动监听，由 `supertest` 管理。

**改动文件**: 新增 `tests/api.test.js`

---

### 4.2 `app.js` 导出改造（为测试准备）

**问题**: 当前 `app.js` 直接调用 `app.listen()` 启动服务，无法被测试框架复用。

**解决方案**: 分离导出和启动：

```javascript
// 底部改为
module.exports = app; // 导出 app 供测试使用

// 仅当直接运行时才启动监听
if (require.main === module) {
  app.listen(port, () => {
    console.log(`工具已启动:http://localhost:${port}`);
  });
}
```

**改动文件**: `app.js`
